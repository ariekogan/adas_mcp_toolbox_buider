# Scheduling Agent — MCP Server

Generated by ADAS MCP Toolbox Builder.

> Users need an easy way to check availability, book meetings, reschedule, and cancel appointments without the back-and-forth of manual calendar coordination.

## Tools

1. **calendar.availability.check** — Check available time slots for a given date range.
   - Inputs: `date_start` (string, **required**), `date_end` (string, optional), `participant_email` (string, optional)
2. **calendar.event.list** — List calendar events for a date range.
   - Inputs: `date_start` (string, **required**), `date_end` (string, optional), `query` (string, optional)
3. **calendar.event.create** — Create a new calendar event on Google Calendar.
   - Inputs: `title` (string, **required**), `start_time` (string, **required**), `end_time` (string, **required**), `attendees` (array, optional), `description` (string, optional)
4. **calendar.event.update** — Update an existing calendar event.
   - Inputs: `event_id` (string, **required**), `title` (string, optional), `start_time` (string, optional), `end_time` (string, optional), `attendees` (array, optional)
5. **calendar.event.delete** — Delete (cancel) a calendar event.
   - Inputs: `event_id` (string, **required**), `notify_attendees` (boolean, optional)

## Quick Start

### Run Locally

```bash
pip install -r requirements.txt
python mcp_server.py
```

### Run with Docker

```bash
docker build -t scheduling-agent-mcp .
docker run -p 8000:8000 scheduling-agent-mcp
```

### Add to Claude Desktop

Add to `~/.claude/config.json`:

```json
{
  "mcpServers": {
    "scheduling-agent": {
      "command": "python",
      "args": ["/path/to/mcp_server.py"]
    }
  }
}
```

## Making Connectors UI-Enabled in ADAS Admin

Each `connector.yaml` includes a `ui` section that defines how the connector
appears in the ADAS Admin settings panel:

```yaml
ui:
  icon: mail              # Icon name for the connector card
  color: "#EA4335"        # Brand color
  category: communication # Grouping in the connector catalog
  settings_schema:        # Form fields shown in Admin UI
    - field: EMAIL_USER
      type: string        # string | password | number | boolean
      required: true
      label: Gmail Address
      placeholder: you@gmail.com
      hint: Your full Gmail email address
```

The `settings_schema` array tells ADAS Admin how to render a configuration
form. Users fill in the fields, and ADAS Core stores the values as encrypted
credentials mapped to the connector's environment variables.
